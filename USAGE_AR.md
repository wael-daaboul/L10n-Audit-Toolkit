# دليل أوامر ومفاتيح تشغيل L10n Audit Toolkit

يحتوي هذا الدليل على جميع الأوامر والمفاتيح (خيارات التشغيل - Flags) المتاحة في أداة `l10n-audit` عبر واجهة سطر الأوامر (CLI) مع شرح مفصل باللغة العربية لكل منها، لتسهيل التعامل مع الأداة للمستخدم العربي.

---

## 📌 الأوامر الأساسية (Main Commands)

تُكتب هذه الأوامر مباشرة بعد كلمة `l10n-audit`.

| الأمر | الشرح بالعربية |
| :--- | :--- |
| `l10n-audit init` | **تهيئة بيئة العمل:** يقوم بإنشاء مجلد `.l10n-audit` في مسار مشروعك، ويستكشف تلقائياً نوع إطار العمل المستخدم لضبط إعدادات الترجمة الافتراضية المناسبة. |
| `l10n-audit doctor` | **تشخيص المشروع:** فحص بيئة العمل الحالية والتأكد من صحة التكوين (Configuration) وهل تم اكتشاف مسارات الترجمة والقواميس بنجاح. |
| `l10n-audit run` | **أمر التشغيل:** الأمر الأساسي لتشغيل عمليات التدقيق (Audits) وتوليد التقارير. القيمة الافتراضية هي التدقيق الشامل `full` إذا لم تحدد مفتاح `--stage`. |
| `l10n-audit update` | **جلب التحديثات للقواعد:** يحدث القواعد العالمية، الفحوصات، والقواميس الافتراضية إلى مساحة عملك المحلية دون تحديث الأداة بالكامل. |
| `l10n-audit self-update` | **دليل ترقية الأداة:** يعرض التوجيهات والأوامر اللازمة لترقية وتحديث أداة `l10n-audit` لآخر إصدار لها عبر `pipx`. |

## 🔑 مفاتيح أو خيارات التشغيل (Options & Flags)

### 1️⃣ مفاتيح شائعة (تعمل مع كل الأوامر)
| المفتاح | الشرح بالعربية |
| :--- | :--- |
| `--help` أو `-h` | يعمل مع أي أمر لعرض قائمة المساعدة والتعليمات. (مثال: `l10n-audit --help` لمعرفة كل الأوامر، أو `l10n-audit run --help` لمعرفة مفاتيح التشغيل). |
| `--version` | للتحقق من رقم الإصدار الحالي المثبت من الأداة (الإصدار الحالي المستقر: **1.2.2**). |
| `--path <PATH>` | تشغيل الأمر على مجلد مشروع مختلف بدل المجلد الذي تقف فيه الآن. (الافتراضي هو المجلد الحالي `.`). |

### 2️⃣ مفاتيح أمر التهيئة (`init`)
| المفتاح | الشرح بالعربية |
| :--- | :--- |
| `--force` | **إجبار** الأداة على إعادة تهيئة بيئة العمل حتى لو كان المجلد `.l10n-audit` موجوداً مسبقاً (سيتم الكتابة فوق الإعدادات الموجودة). |
| `--channel <CHANNEL>` | تحديد قناة القوالب الافتراضية عند التهيئة: إما `stable` (للنسخة المستقرة الموصى بها - **الافتراضي**)، أو `main` (لآخر التعديلات التطويرية). |
| `--from-github` | سحب قوالب الإعدادات والقواميس من مستودع GitHub بدلاً من الملفات المدمجة داخل الأداة. يستلزم تحديد `--repo`. |
| `--repo <URL>` | يُستخدم مع مفتاح `--from-github` لتحديد رابط مستودع GitHub المخصص الذي تريد سحب القوالب منه. |

### 3️⃣ مفاتيح أمر التحديث (`update`)
| المفتاح | الشرح بالعربية |
| :--- | :--- |
| `--check` | فحص حالة المشروع المحلية وإصدار الأداة المثبَّت، **دون** تطبيق أي تغيير فعلي. مفيد لمعرفة ما إذا كانت هناك تحديثات متاحة. |
| `--channel <CHANNEL>` | تحديد قناة التحديث: إما `stable` (للنسخة المستقرة - **الافتراضي**)، أو `main` (لآخر التعديلات التطويرية). |
| `--from-github` | سحب وتحديث قوالب الإعدادات والقواميس من مستودع GitHub بدلاً من الملفات المدمجة داخل الأداة. يستلزم تحديد `--repo`. |
| `--repo <URL>` | يُستخدم مع مفتاح `--from-github` لتحديد رابط مستودع GitHub المخصص الذي تريد سحب التحديثات منه. |

### 4️⃣ مفاتيح أمر التشغيل والتدقيق (`run`)

الأمر `l10n-audit run` إذا شغّلته بدون أي مفتاح سيعمل تلقائياً بمرحلة `full` (التدقيق الشامل). يمكنك تخصيص المرحلة باستخدام المفتاح `--stage`.

#### مفتاح تحديد المرحلة (Stage)
يستخدم بالشكل التالي `l10n-audit run --stage <NAME>`

| قيمة الـ Stage | الشرح بالعربية |
| :--- | :--- |
| `fast` | **التدقيق السريع (الموصى به يومياً):** يراجع أخطاء الترجمة الأساسية، جودة اللغة العربية، تطابق المتغيرات، وفحص المصطلحات. |
| `full` | **التدقيق الشامل:** الفحص السريع + فحص بنية رسائل التعدد (ICU) + المراجعة النحوية واللغوية المعمقة للإنجليزية عبر LanguageTool. |
| `autofix` | **مولد الإصلاح التلقائي:** وضع لتوليد خطة إصلاح تلقائية آمنة للمسافات وعلامات الترقيم البسيطة دون المساس بالمعاني. يعطيك النتائج لتراجعها قبل التطبيق. |
| `ai-review` | المراجعة باستخدام **الذكاء الاصطناعي**. (انظر مفاتيح الذكاء الاصطناعي أدناه لتفعيله). |
| `placeholders` | اختبار تطابق المتغيرات أو الأقواس (Placeholders) بشكل **منفصل** ومستقل. |
| `terminology` | تشغيل فحص القاموس والمصطلحات الإجبارية (Glossary enforcement) بشكل **منفصل**. |
| `ar-qc` | تشغيل فحص الجودة الخاص بالتنسيق وعلامات الترقيم والمسافات للغة العربية **منفصلاً**. |
| `ar-semantic` | تشغيل الفحص الدلالي العربي **منفصلاً** لاقتراح تحسينات صياغية وإصلاح تشوه المعنى. |
| `grammar` | تشغيل أداة LanguageTool لفحص نحو وإملاء اللغة الإنجليزية **بمفردها**. |
| `icu` | فحص تطابق وهيكلية رسائل (ICU Messages) (لأشياء مثل الترجمات المتعددة بصيغ المفرد والجمع) بشكل **منفصل**. |
| `reports` | إعادة تجميع وتوليد ملفات التقارير من النتائج المتوفرة مسبقاً دون إعادة التدقيق من الصفر لكل شيء. |

#### مفاتيح أداة الذكاء الاصطناعي (AI Options)
تعمل هذه المفاتيح فقط عند استخدامك لمرحلة التدقيق بالذكاء الاصطناعي هكذا: `l10n-audit run --stage ai-review`

| المفتاح | الشكل أو القيمة المطلوبة | الشرح بالعربية |
| :--- | :--- | :--- |
| `--ai-enabled` | `--ai-enabled` فقط | **مفتاح التفعيل الإلزامي:** يجب وضعه لتمكين عمل ميزات الذكاء الاصطناعي بالمشروع لتجنب التشغيل بالخطأ. |
| `--ai-api-key` | `"sk-..."` | **مفتاح الـ API** الخاص بحسابك في OpenAI أو OpenRouter (يُفضل حفظه كمتغير بيئة `export OPENROUTER_API_KEY` للحماية بدل تمريره كأمر). |
| `--ai-model` | `"openai/gpt-4o-mini"` | **تحديد النموذج (Model):** اسم نموذج الذكاء الاصطناعي المراد استخدامه. |
| `--ai-api-base` | `"https://openrouter.ai/api/v1"` | (مفتاح اختياري) لتغيير الرابط الأساسي للاتصال بالنماذج في حال كنت تستخدم مزوداً آخر (سوى OpenAI الرسمي) مثل OpenRouter أو محرك محلي مثل Ollama. |
| `--write-reports`| `True`/`False` | تحديد ما إذا كنت تريد توليد ملفات التقارير التفصيلية لكل أداة (الافتراضي: `True`). |

## 🌐 خادم الواجهة البرمجية المحلي (Local HTTP API)

توفر الأداة الآن خادم API محلياً مبنيًا على FastAPI يتيح لك تشغيل عمليات التدقيق عبر طلبات HTTP، وهو مفيد لدمج الأداة في واجهات المستخدم الرسومية أو تطبيقات الويب.

**كيفية التشغيل:**
1. انتقل لمجلد `http_api`.
2. شغّل الخادم: `uvicorn server:app --reload`.
3. تصفح التوثيق التفاعلي (Swagger) عبر: `http://localhost:8000/docs`.

**المميزات:**
* تشغيل غير حاصر (Non-blocking) للعمليات الطويلة.
* دعم كامل لجميع مراحل التدقيق.
* معالجة أخطاء محسنة للإصدار **1.2.1**.

---

> 💡 **نصيحة عامة:** كل المخرجات، سواء كانت تقارير أخطاء، جداول مراجعة للمترجم، نصائح ذكاء اصطناعي، أو ملفات التصحيح الآلية، ستجدها دائماً مرتبة وجاهزة داخل مجلد `Results/` في مسار مشروعك بعد تشغيل الأمر متضمناً مرحلة الفحص المُرادة.
