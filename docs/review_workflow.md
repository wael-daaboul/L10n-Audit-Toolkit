# Review Workflow

The L10n Audit Toolkit enforces a strict separation between machine-discovered issues, human review, and executable fixes. The workflow operates as a continuous pipeline:

### `Audit → Review Queue → Human Decisions → Prepare Apply → Apply → Reconcile`

---

## 1. Audit (Machine Discovery)
```bash
l10n-audit run --stage fast
# or
l10n-audit run --stage ai-review
```
During the audit stage, the toolkit scans your locales using fast regex rules or an optional AI engine. It detects terminology mismatches, missing placeholders, and poor sentiment. *It does not modify your project files.* The findings are instead aggregated into the master state.

## 2. Review Queue (Generation)
The master state is projected into a human-friendly spreadsheet located at `Results/review/review_queue.xlsx`. This file contains all detected issues along with a suggested fix (either generated by a rule or suggested by the AI).

## 3. Human Decisions (The Human-in-the-Loop)
A translator or localization manager opens `review_queue.xlsx`.
For each suggestion they agree with, they:
1. Review the `current_value` and `candidate_value` columns.
2. Adjust `candidate_value` or add `review_note` if needed.
3. Change the `status` column from `pending` to `approved`.

## 4. Prepare Apply (Freeze the Execution Contract)
```bash
l10n-audit prepare-apply
```
This command reads `Results/review/review_queue.xlsx`, validates every row strictly, rejects unsafe rows, and writes `Results/review/review_final.xlsx`.

`prepare-apply` transforms the editable review workspace into a deterministic, execution-safe contract.

`review_final.xlsx` is a frozen, execution-safe workbook generated by `prepare-apply`. It is immutable and must not be edited manually.

If `review_queue.xlsx` changes after freezing, run `l10n-audit prepare-apply` again to regenerate `review_final.xlsx`.

| File | Role |
| :--- | :--- |
| `review_queue.xlsx` | Editable human review workspace |
| `review_final.xlsx` | Frozen execution contract |

## 5. Apply (Safe Integration)
```bash
l10n-audit apply
```
The toolkit reads only `Results/review/review_final.xlsx`. It does not read `review_queue.xlsx` directly. Before making any changes, it verifies the original translation hasn't changed (using state hashes). If safe, it generates isolated `.fix` files formatted exactly for your framework in your original project directory.

## 6. Reconcile (Recording the Truth)
Finally, after fixes are applied, the toolkit updates `audit_master.json`, stamping the applied items as `applied`. This prevents the same translation from appearing in tomorrow's review queue.

---

## Optional: Fully Automated Mode
If you trust the engine entirely (not recommended for production semantics without human review), you can bypass the manual spreadsheet step:
```bash
l10n-audit apply --all
```
This forces all rows in `review_final.xlsx` to be applied, but you must still run `prepare-apply` first.
